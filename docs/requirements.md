# 拼豆智能助手 - 需求文档

> 版本：v0.1.0 · 最后更新：2026-06-22 · 状态：规划中

## 1. 项目概述

| 项 | 内容 |
|----|------|
| **项目名** | 拼豆智能助手（Pindou Assistant，暂定） |
| **一句话定位** | AI 驱动的拼豆图纸生成与进度追踪工具 |
| **目标用户** | 拼豆爱好者（个人作者、亲子手工、设计师） |
| **核心价值** | ① 任意图片秒变拼豆图纸；② 拼豆过程中 AI 辅助识别进度，告诉你还差什么 |
| **商业模式** | 免费开源（AGPL-3.0），未来可选增值服务 |
| **多端策略** | 微信小程序 + H5 双端，一套代码（Taro） |

## 2. 用户痛点

1. **找图纸难** — 网上图片不一定是像素图，自己画太麻烦
2. **配色难** — 不知道一张图该用哪些色号、用量多少
3. **豆板尺寸不匹配** — 算不清一张图需要几块板
4. **拼到一半不知道进度** — 拼错了不知道、漏拼了不知道
5. **色号体系混乱** — MARD / COCO / 漫漫 / 盼盼 / 咪小窝 各不相同，互不通用

## 3. 核心功能（按优先级）

### 3.1 P0 — MVP（必须有，预计 2-4 周）

> 💡 **颜色合并（F10）是生成质量第一优先级**：竞品 [pindou.org](https://pindou.org/) 的核心卖点之一。它直接决定图纸「成品感」和采购清单的色号数（影响备料成本），属纯算法（CIEDE2000 + BFS，客户端可完成），原归入 P1「AI 增强」分类有误，提前到 P0。详见 [competitive-analysis.md](./competitive-analysis.md)。

| ID | 功能 | 描述 | 验收标准 |
|----|------|------|---------|
| F01 | **图片上传** | 支持相册、拍照、拖拽上传，自动 EXIF 旋转校正 | JPG/PNG/WebP ≤ 10MB |
| F02 | **像素化处理** | 按目标尺寸（29×29 / 58×58 等）将原图像素化，主导色提取算法 | 处理一张 1024px 图 < 1s |
| F03 | **色号映射** | 基于色差将像素映射到选定品牌色号体系（MVP 先欧氏 RGB，随即升级 CIEDE2000） | 支持 5 大品牌 |
| F04 | **品牌切换** | MARD / COCO / 漫漫 / 盼盼 / 咪小窝 一键切换 | 切换后实时重新映射 |
| F05 | **色板规格** | 支持 96 色 / 144 色 / 168 色 / 全 291 色多种色板 | 用户可选色板规模 |
| F06 | **图纸导出** | 导出带网格线和色号标注的 PNG，可直接打印 | A4 打印清晰可读 |
| F07 | **采购清单** | 自动统计各色号用量，导出清单图（含色号、色块、数量） | 一图看清所需豆子 |
| F08 | **豆板用量** | 计算所需豆板数量（按 29×29 单元），标注拼接位置 | 准确无误 |
| F10 | **颜色合并** | 两种模式：①空间孤立噪点 BFS 清除 ②全局相近色号归并；基于 CIEDE2000 色差；支持「最多 N 色」限制 | 默认参数下色号数大幅减少（如 38→12 色）且肉眼无明显差异；阈值 / N 色可调、实时预览 |

### 3.2 P1 — AI 增强（预计 4-8 周）

| ID | 功能 | 描述 | 验收标准 |
|----|------|------|---------|
| F09 | **AI 抠图** | 自动移除背景（边界洪水填充 + U2Net 模型） | 复杂背景也能抠干净 |
| F11 | **颜色排除** | 一键排除没有的色号，自动重映射到最近似色 | 实时重映射 |
| F12 | **手动精修** | 单格点击修改色号，撤销/重做 | 支持 50 步撤销 |
| F13 | **智能配色建议** | 根据图片内容推荐色板组合（CIEDE2000 色差算法） | 推荐 ≤ 48 色 |
| F14 | **多尺寸预览** | 同一张图实时预览不同尺寸（29×29 / 50×50 / 100×100） | 切换 < 500ms |

### 3.3 P2 — 创新能力（预计 8-16 周，差异化卖点）★

| ID | 功能 | 描述 | 验收标准 |
|----|------|------|---------|
| F15 | **拍照识别进度** | 用户拍下未完成的豆板，AI 对比设计图纸，识别已完成/未完成/拼错的位置 | 正俯视光照良好场景准确率 > 90% |
| F16 | **进度提示** | 列出「还差什么色号 ×N 颗」「哪里拼错了，应该是什么色」 | 文字 + 高亮图双重展示 |
| F17 | **AR 辅助**（远期） | 实时摄像头叠加，直接在屏幕上看到每格该放什么色 | 实时帧率 ≥ 15fps |

### 3.4 P3 — 生态能力（远期）

| ID | 功能 | 描述 |
|----|------|------|
| F18 | **AI 生图转拼豆** | 输入文字描述（"画一只赛博朋克猫"）→ SDXL 生图 → 自动转拼豆图纸 |
| F19 | **社区分享** | 图纸广场、作品秀、模板复用 |
| F20 | **个人作品库** | 收藏图纸、记录进度、历史作品 |

## 4. 非功能需求

| 维度 | 要求 |
|------|------|
| **多端** | 微信小程序 + H5（一套代码，Taro 编译） |
| **性能** | 单张图处理 < 3s（标准 29×29）；进度识别 < 5s |
| **离线** | 色号库、豆板规格本地缓存，无网也能用基础功能 |
| **国际化** | 中文优先，预留 i18n 接口 |
| **无障碍** | WCAG AA 标准，色弱模式 |
| **兼容性** | iOS 12+ / Android 7+ / 微信基础库 2.27+ |
| **数据隐私** | 图片处理默认在客户端/小程序端完成，不上云；AI 增强功能需明确告知 |
| **协议** | AGPL-3.0（衍生作品必须开源） |

## 5. 用户故事（核心场景）

### 故事 1：新手第一次做拼豆
> 小红看到朋友拼了个皮卡丘，也想试试。她打开「拼豆智能助手」小程序，上传了一张皮卡丘图片。系统自动像素化、推荐用 MARD 5mm 豆 + 29×29 板。小红一键导出图纸和色号清单，直接照着淘宝买齐材料。

### 故事 2：拼到一半忘了进度 ★
> 小明拼了 3 天的塞尔达，中间出差一周忘了拼到哪。回来后拍张照，系统说：「第 4 行第 7 列拼错了，应该是 MARD A4 不是 A6；还差 F2 红色 12 颗、H7 黑色 8 颗」。小明照着提示继续拼。

### 故事 3：AI 帮我创作
> 小华想送朋友一个独一无二的礼物，输入「一只抱着爱心的小柴犬，像素风」。AI 生图后自动转成拼豆图纸，直接下单。

### 故事 4：老手精细调整
> 老玩家觉得自动配色不满意，手动排除掉手里没有的 5 个色号，系统自动重映射到最接近的替代色；再手动精修几个格子，最后导出。

## 6. 不做的事（Out of Scope）

- ❌ **不做拼豆销售** — 导流到第三方电商即可
- ❌ **不做 3D 立体拼豆图纸** — 仅平面
- ❌ **不做拼豆加工服务** — 不代拼
- ❌ **不做社交聊天** — 仅作品分享

## 8. 附录

### 8.1 术语表

- **拼豆（Perler Bead）**：一种用塑料豆子在钉板上拼出图案、加热融合的手工艺术
- **色号**：每个品牌对每种颜色的唯一编号
- **豆板**：带钉子的板子，豆子插在上面，常见规格 29×29 孔
- **像素化**：将图片降采样到指定分辨率的网格图
- **CIEDE2000**：工业标准的色差算法，比欧氏距离更符合人眼感知
